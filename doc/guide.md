# 数据抓取流程

## 整理 API

整理 API 的规则，需要根据 API 的更新频率，分为以下几种：

- **固定频率 API**：初始抓取一次，之后不再抓取；命名规则为：`{命名空间}.{赛季 ID}.json`
- **每日更新 API**：每日抓取一次，命名规则为：`{命名空间}.{赛季 ID}.{日期}.json`
- **无赛季 ID API**：文件名中不包含赛季 ID，如 `{命名空间}.json` 或 `{命名空间}.{日期}.json`

## 自动抓取流程（一级抓取）

1. 首先查询赛季列表信息，查询到：`"project": "KPL", "is_latest": 1 ` 的对象，就是 KPL 最新赛季。
2. 获取赛季信息，赛季信息中包含赛季的开始和结束日期。
3. 根据开始和结束日期确定是否执行抓取任务：如果在日期范围内（包括结束日期的后 3 天），则执行抓取任务；否则不执行抓取任务。
4. 从 `player-career-wuyan.{日期}.json` 文件中读取：
   - 选手名称（如 `KSG.无言`）
   - 战队名称（如 `KSG`）
5. 将赛季 ID 替换所有 API 中的 `{season_id}` 参数，将战队名称替换 `{team_name}` 参数。
6. 只关注 "无言" 选手的信息；一些抓取全部选手的 API，需要从响应数据中筛选出 "无言" 的选手数据，单独保存至文件。
7. 保存文件的格式为 JSON，命名规范为：`{命名空间}.{赛季 ID}.{日期}.json`。

## 二级抓取：英雄对局详情

在一级抓取完成后，执行二级抓取，获取每个英雄的详细对局数据：

1. 从 `player-hero-summary.{赛季 ID}.{日期}.json` 文件中提取本赛季使用的所有英雄名称。
   - 该文件包含选手在本赛季使用过的全部英雄及其胜场统计，数据更准确。
2. 遍历英雄列表，对每个英雄调用 `player-hero-battles` API：
   - 请求参数：`player_name`、`hero_name`、`season`
   - 获取该英雄的所有对局详情（出装、铭文、阵容、KDA 等）
3. 统计每个英雄的总场次、胜场、负场。
4. 保存至 `player-hero-battles.{赛季 ID}.{日期}.json` 文件。
