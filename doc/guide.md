# 数据抓取流程

## 整理API

整理API的规则，需要根据API的更新频率，分为以下几种：

- 固定频率、指定不更新的API： 初始抓取一次，之后不再抓取；命名规则为： `{命名空间}{赛季ID}.json`
- 不固定频率、指定更新的API： 每日抓取一次，命名规则为： `{命名空间}{赛季ID}.{日期}.json`

## 自动抓取流程

1. 首先查询赛季列表信息，查询到： `"project": "KPL", "is_latest": 1 ` 的对象，就是KPL最新赛季。
2. 获取赛季信息，赛季信息中包含赛季的开始和结束日期。
3. 根据开始和结束日期确定是否执行抓取任务：如果在日期范围内（包括结束日期的第二天），则执行抓取任务；否则不执行抓取任务。
4. 将赛季ID替换列举的所有 API 中 "season_id" 参数，将 "team_name" 替换为 "KSG"（目前无言所在战队名称）。
5. 只关注 "无言" 选手的信息；一些抓取全部选手的 API，需要从响应数据中筛选出 "无言" 的选手数据，单独保存至文件。
6. 保存文件的格式为 JSON ，命名规范为： `{命名空间}{赛季ID}.{日期}.json`。
